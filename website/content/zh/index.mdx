模块化、可扩展的微信 iLink Bot SDK（Node.js）—— 支持中间件管道、完整媒体收发、可插拔存储和会话自动恢复。**零运行时依赖**，TypeScript 优先，生产级品质。灵感来源于 [tencent-weixin/openclaw-weixin-cli](https://github.com/nicepkg/openclaw-weixin)。[GitHub](https://github.com/anthropic/wechatbot)。

你的 Agent 通过 iLink Bot 协议连接微信 —— 二维码扫码登录、长轮询收消息、HTTP 发消息。SDK 处理**所有复杂逻辑**：`context_token` 生命周期、AES-128-ECB 媒体加密、输入状态指示器、会话恢复和智能文本分片。

<Section id="quick-start" title="快速开始">

**三步**即可运行微信机器人。

1. 安装 SDK：

```bash
npm install @wechatbot/wechatbot
```

2. 创建你的机器人：

```typescript
import { WeChatBot } from '@wechatbot/wechatbot'

const bot = new WeChatBot()
await bot.login()

bot.onMessage(async (msg) => {
  await bot.sendTyping(msg.userId)
  await bot.reply(msg, `Echo: ${msg.text}`)
})

await bot.start()
```

3. 在微信中扫描终端显示的二维码。完成 —— 你的机器人已上线。

凭证会**自动保存**到 `~/.wechatbot/`。下次重启时，机器人跳过扫码直接重连。如果会话过期，它会**自动重新认证**并显示新的二维码。

<Caption>零配置。零 Webhook。完全在你的机器上运行。</Caption>

</Section>

<Section id="how-it-works" title="工作原理">

微信 iLink Bot 协议使用**二维码登录** + **HTTP 长轮询**收消息。流程如下：

<CodeBlock lang="diagram" lineHeight="1.3" showLineNumbers={false}>
{`┌─────────────┐     ┌──────────────────┐     ┌─────────────────┐
│  微信用户    │     │   iLink 服务器   │     │   你的机器人    │
└──────┬───────┘     └────────┬─────────┘     └────────┬────────┘
       │                      │                        │
       │                      │   GET /get_bot_qrcode  │
       │                      │<───────────────────────│
       │                      │───────────────────────>│ 二维码 URL
       │   微信扫码           │                        │
       │─────────────────────>│                        │
       │                      │   status: confirmed    │
       │                      │───────────────────────>│ bot_token
       │                      │                        │
       │  "你好"              │                        │
       │─────────────────────>│                        │
       │                      │  POST /getupdates      │
       │                      │<───────────────────────│
       │                      │───────────────────────>│ 消息 + context_token
       │                      │                        │
       │                      │  POST /sendtyping      │
       │                      │<───────────────────────│
       │  "对方正在输入..."    │                        │
       │<─────────────────────│                        │
       │                      │                        │
       │                      │  POST /sendmessage     │
       │                      │<───────────────────────│ context_token + 回复
       │  "Echo: 你好"        │                        │
       │<─────────────────────│                        │
       │                      │                        │`}
</CodeBlock>

协议中最关键的概念是 `context_token` —— 每个回复**必须**回传入站消息中的 token。SDK 自动管理这个过程，按用户缓存 token 并在重启后持久化。

</Section>

<Section id="middleware" title="中间件">

中间件管道是 **Express/Koa 风格**的链式处理，在消息处理器之前运行。每个中间件可以转换、过滤或短路消息。

```typescript
import {
  WeChatBot,
  loggingMiddleware,
  rateLimitMiddleware,
  typeFilterMiddleware,
  filterMiddleware,
} from '@wechatbot/wechatbot'

const bot = new WeChatBot()

// 内置中间件
bot.use(loggingMiddleware(bot.logger))           // 记录每条消息
bot.use(rateLimitMiddleware({                     // 按用户限流
  maxMessages: 10, windowMs: 60_000
}))
bot.use(typeFilterMiddleware('text', 'image'))    // 只处理文本和图片
bot.use(filterMiddleware(/^\/\w+/))              // 只处理 /命令
```

编写**自定义中间件** —— 解析命令、注入上下文、测量耗时：

```typescript
// 命令解析器中间件
bot.use(async (ctx, next) => {
  const text = ctx.message.text.trim()
  if (text.startsWith('/')) {
    const [cmd, ...args] = text.slice(1).split(/\s+/)
    ctx.state.set('command', cmd?.toLowerCase())
    ctx.state.set('args', args)
  }
  await next()
})

// 响应计时器
bot.use(async (ctx, next) => {
  const start = Date.now()
  await next()
  console.log(`处理耗时 ${Date.now() - start}ms`)
})
```

不调用 `next()` 的中间件会**停止链式执行** —— 非常适合用于认证检查、垃圾消息过滤或命令路由。

<Caption>可组合。可测试。每个中间件都是纯函数。</Caption>

</Section>

<Section id="media" title="媒体处理">

完整支持**图片、文件、语音和视频** —— 下载和上传。SDK 透明地处理 AES-128-ECB 加密、CDN 通信和双密钥格式解码。

```typescript
bot.onMessage(async (msg) => {
  // 从收到的消息中下载图片
  if (msg.type === 'image' && msg.images[0]?.media) {
    const data = await bot.downloadMedia(
      msg.images[0].media,
      msg.images[0].aeskey
    )
    await writeFile('photo.jpg', data)
    await bot.reply(msg, `已下载 ${data.length} 字节`)
  }

  // 语音消息包含转写文本
  if (msg.type === 'voice' && msg.voices[0]?.text) {
    await bot.reply(msg, `你说了: "${msg.voices[0].text}"`)
  }

  // 文件下载
  if (msg.type === 'file' && msg.files[0]?.media) {
    const fileData = await bot.downloadMedia(msg.files[0].media)
    await bot.reply(msg, `收到: ${msg.files[0].fileName}`)
  }
})
```

向用户上传文件：

```typescript
import { MediaType } from '@wechatbot/wechatbot'

const result = await bot.sendMedia(userId, {
  data: imageBuffer,
  userId,
  mediaType: MediaType.IMAGE,
})
```

CDN 加密层自动处理两种密钥格式 —— `base64(原始 16 字节)` 和 `base64(十六进制字符串)`，以及 `image_item.aeskey` 中的直接十六进制字符串。

<Caption>AES-128-ECB。双密钥格式。CDN 上传/下载。全部透明处理。</Caption>

</Section>

<Section id="storage" title="可插拔存储">

所有 SDK 状态 —— 凭证、轮询游标、context token、typing ticket —— 都通过**可插拔存储接口**持久化。无需修改任何机器人代码即可切换后端。

```typescript
import { WeChatBot, FileStorage, MemoryStorage } from '@wechatbot/wechatbot'

// 文件存储（默认）—— 重启后保留
const bot1 = new WeChatBot({ storage: 'file' })

// 内存存储 —— 快速，临时
const bot2 = new WeChatBot({ storage: 'memory' })

// 自定义存储 —— 实现 5 个方法
class RedisStorage {
  async get(key) { /* redis.get */ }
  async set(key, value) { /* redis.set */ }
  async delete(key) { /* redis.del */ }
  async has(key) { /* redis.exists */ }
  async clear() { /* redis.flushdb */ }
}

const bot3 = new WeChatBot({ storage: new RedisStorage() })
```

文件存储将每个键保存为 `~/.wechatbot/` 中的 JSON 文件，权限为 `0600`。context token 在每批消息处理后刷新到磁盘，因此能在进程重启后恢复。

</Section>

<Section id="events" title="事件">

机器人会发出**类型化事件**用于生命周期监控。在 TypeScript 中提供完整的 IntelliSense。

```typescript
bot.on('login', (creds) => {
  console.log(`已登录: ${creds.accountId}`)
})

bot.on('session:expired', () => {
  console.log('会话过期 —— 正在自动重新登录')
})

bot.on('session:restored', (creds) => {
  console.log(`已恢复在线: ${creds.accountId}`)
})

bot.on('error', (err) => {
  alerting.send(`机器人错误: ${err}`)
})

bot.on('poll:start', () => console.log('开始轮询'))
bot.on('poll:stop', () => console.log('停止轮询'))
bot.on('close', () => console.log('机器人已关闭'))
```

会话恢复是**完全自动的** —— 当收到 `-14`（会话超时）错误时，SDK 会清除所有状态，重新执行二维码登录流程，然后恢复轮询。无需人工干预。

</Section>

对于包含多个项目的复杂消息，使用**流式构建器 API**：

  .text("这是你的报告：")
  .file({
    media: cdnMediaRef,
    fileName: 'Q1-报告.pdf',
    size: 542188,

<Section id="protocol" title="协议概览">

iLink Bot API 基于 HTTP/JSON，核心端点如下：

- **`GET /get_bot_qrcode`** —— 请求登录二维码
- **`GET /get_qrcode_status`** —— 轮询扫码状态 → confirmed → 获取 `bot_token`
- **`POST /getupdates`** —— 长轮询接收消息（35 秒挂起）
- **`POST /sendmessage`** —— 发送文本或媒体消息
- **`POST /getconfig`** —— 获取用户的 `typing_ticket`
- **`POST /sendtyping`** —— 显示/隐藏"对方正在输入..."
- **`POST /getuploadurl`** —— 请求 CDN 上传参数
- **`POST CDN /upload`** —— 上传 AES 加密的媒体文件
- **`GET CDN /download`** —— 下载 AES 加密的媒体文件

每个 POST 请求需要：`Authorization: Bearer <bot_token>`、`AuthorizationType: ilink_bot_token`，以及一个随机的 `X-WECHAT-UIN` 头。所有请求体包含 `base_info: { channel_version: "2.0.0" }`。

最关键的概念是 `context_token` —— 它**不是可选的**。没有它，回复无法路由到正确的微信会话。SDK 按 `(accountId, userId)` 对缓存它并持久化到存储。

</Section>

<Section id="security" title="安全性与隐私">

一切运行在**你的机器上**。没有远程服务器、没有账户、没有遥测。

- **仅限本地** —— 所有 API 调用直接从你的机器发送到 `ilinkai.weixin.qq.com`
- **凭证静态加密** —— 以 `0600` 权限存储在 `~/.wechatbot/`
- **无代理或中继** —— SDK 直接与微信 iLink 服务器通信
- **媒体加密** —— 所有 CDN 上传/下载使用 AES-128-ECB；文件在上传前本地加密
- **会话隔离** —— 每个机器人实例独立管理自己的凭证和状态

</Section>
