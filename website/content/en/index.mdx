A modular, extensible WeChat iLink Bot SDK for Node.js вҖ” with middleware pipeline, full media support, pluggable storage, and automatic session recovery. **Zero runtime dependencies**, TypeScript-first, production-grade. Inspired by [tencent-weixin/openclaw-weixin-cli](https://github.com/nicepkg/openclaw-weixin). [GitHub](https://github.com/anthropic/wechatbot).

Your agent connects to WeChat via the iLink Bot protocol вҖ” QR code login, long-poll message receiving, and HTTP message sending. The SDK handles **all the complexity**: `context_token` lifecycle, AES-128-ECB media encryption, typing indicators, session recovery, and smart text chunking.

<Section id="quick-start" title="Quick start">

**Three steps** to a working WeChat bot.

1. Install the SDK:

```bash
npm install @wechatbot/wechatbot
```

2. Create your bot:

```typescript
import { WeChatBot } from '@wechatbot/wechatbot'

const bot = new WeChatBot()
await bot.login()

bot.onMessage(async (msg) => {
  await bot.sendTyping(msg.userId)
  await bot.reply(msg, `Echo: ${msg.text}`)
})

await bot.start()
```

3. Scan the QR code in WeChat when prompted. Done вҖ” your bot is live.

Credentials are **automatically saved** to `~/.wechatbot/`. On next restart, the bot skips the QR scan and reconnects instantly. If the session expires, it **automatically re-authenticates** with a new QR code.

<Caption>Zero config. Zero webhooks. Runs entirely on your machine.</Caption>

</Section>

<Section id="middleware" title="Middleware">

The middleware pipeline is an **Express/Koa-style** chain that runs before message handlers. Each middleware can transform, filter, or short-circuit messages.

```typescript
import {
  WeChatBot,
  loggingMiddleware,
  rateLimitMiddleware,
  typeFilterMiddleware,
  filterMiddleware,
} from '@wechatbot/wechatbot'

const bot = new WeChatBot()

// Built-in middleware
bot.use(loggingMiddleware(bot.logger))           // Log every message
bot.use(rateLimitMiddleware({                     // Rate limit per user
  maxMessages: 10, windowMs: 60_000
}))
bot.use(typeFilterMiddleware('text', 'image'))    // Only text & images
bot.use(filterMiddleware(/^\/\w+/))              // Only /commands
```

Write **custom middleware** вҖ” parse commands, inject context, measure timing:

```typescript
// Command parser middleware
bot.use(async (ctx, next) => {
  const text = ctx.message.text.trim()
  if (text.startsWith('/')) {
    const [cmd, ...args] = text.slice(1).split(/\s+/)
    ctx.state.set('command', cmd?.toLowerCase())
    ctx.state.set('args', args)
  }
  await next()
})

// Response timer
bot.use(async (ctx, next) => {
  const start = Date.now()
  await next()
  console.log(`Processed in ${Date.now() - start}ms`)
})
```

Middleware that doesn't call `next()` **stops the chain** вҖ” perfect for auth checks, spam filters, or command routing.

<Caption>Composable. Testable. Each middleware is a pure function.</Caption>

</Section>

<Section id="media" title="Media handling">

Full support for **images, files, voice, and video** вҖ” both download and upload. The SDK handles AES-128-ECB encryption, CDN communication, and dual key format decoding transparently.

```typescript
bot.onMessage(async (msg) => {
  // Download image from incoming message
  if (msg.type === 'image' && msg.images[0]?.media) {
    const data = await bot.downloadMedia(
      msg.images[0].media,
      msg.images[0].aeskey
    )
    await writeFile('photo.jpg', data)
    await bot.reply(msg, `Downloaded ${data.length} bytes`)
  }

  // Voice messages include transcription
  if (msg.type === 'voice' && msg.voices[0]?.text) {
    await bot.reply(msg, `You said: "${msg.voices[0].text}"`)
  }

  // File download
  if (msg.type === 'file' && msg.files[0]?.media) {
    const fileData = await bot.downloadMedia(msg.files[0].media)
    await bot.reply(msg, `Got: ${msg.files[0].fileName}`)
  }
})
```

Upload files to users:

```typescript
import { MediaType } from '@wechatbot/wechatbot'

const result = await bot.sendMedia(userId, {
  data: imageBuffer,
  userId,
  mediaType: MediaType.IMAGE,
})
```

The CDN crypto layer handles both key formats automatically вҖ” `base64(raw 16 bytes)` and `base64(hex string)`, plus direct hex strings from `image_item.aeskey`.

<Caption>AES-128-ECB. Dual key format. CDN upload/download. All transparent.</Caption>

</Section>

<Section id="storage" title="Pluggable storage">

All SDK state вҖ” credentials, poll cursors, context tokens, typing tickets вҖ” is persisted through a **pluggable storage interface**. Swap backends without changing any bot code.

```typescript
import { WeChatBot, FileStorage, MemoryStorage } from '@wechatbot/wechatbot'

// File storage (default) вҖ” survives restarts
const bot1 = new WeChatBot({ storage: 'file' })

// Memory storage вҖ” fast, ephemeral
const bot2 = new WeChatBot({ storage: 'memory' })

// Custom storage вҖ” implement 5 methods
class RedisStorage {
  async get(key) { /* redis.get */ }
  async set(key, value) { /* redis.set */ }
  async delete(key) { /* redis.del */ }
  async has(key) { /* redis.exists */ }
  async clear() { /* redis.flushdb */ }
}

const bot3 = new WeChatBot({ storage: new RedisStorage() })
```

File storage saves each key as a JSON file in `~/.wechatbot/` with `0600` permissions. Context tokens are flushed to disk after each message batch, so they survive process restarts.

</Section>

<Section id="events" title="Events">

The bot emits **typed events** for lifecycle monitoring. Full IntelliSense in TypeScript.

```typescript
bot.on('login', (creds) => {
  console.log(`Logged in: ${creds.accountId}`)
})

bot.on('session:expired', () => {
  console.log('Session expired вҖ” re-login happening automatically')
})

bot.on('session:restored', (creds) => {
  console.log(`Back online: ${creds.accountId}`)
})

bot.on('error', (err) => {
  alerting.send(`Bot error: ${err}`)
})

bot.on('poll:start', () => console.log('Polling started'))
bot.on('poll:stop', () => console.log('Polling stopped'))
bot.on('close', () => console.log('Bot shut down'))
```

Session recovery is **fully automatic** вҖ” when a `-14` (session timeout) error is received, the SDK clears all state, re-runs the QR login flow, and resumes polling. No manual intervention needed.

</Section>

<Section id="message-builder" title="Message builder">

For complex messages with multiple items, use the **fluent builder API**:

```typescript
const msg = bot.createMessage(userId)
  .text("Here's your report:")
  .file({
    media: cdnMediaRef,
    fileName: 'Q1-report.pdf',
    size: 542188,
  })
  .build()

await bot.sendMessage(msg)
```

Long text is **automatically chunked** at natural boundaries вҖ” paragraphs first, then line breaks, then spaces. Each chunk gets its own `client_id` but shares the same `context_token`.

```typescript
// Smart chunking вҖ” splits a 5000-char text into ~3 messages
// Priority: paragraph break вҶ’ line break вҶ’ space вҶ’ hard cut at 2000
await bot.reply(msg, veryLongText)
```

<Section id="protocol" title="Protocol overview">

The iLink Bot API is HTTP/JSON based with these core endpoints:

- **`GET /get_bot_qrcode`** вҖ” request a QR code for login
- **`GET /get_qrcode_status`** вҖ” poll for scan вҶ’ confirmed вҶ’ get `bot_token`
- **`POST /getupdates`** вҖ” long-poll for incoming messages (35s hold)
- **`POST /sendmessage`** вҖ” send text or media messages
- **`POST /getconfig`** вҖ” get `typing_ticket` for a user
- **`POST /sendtyping`** вҖ” show/hide "typing..." indicator
- **`POST /getuploadurl`** вҖ” request CDN upload parameters
- **`POST CDN /upload`** вҖ” upload AES-encrypted media
- **`GET CDN /download`** вҖ” download AES-encrypted media

Every POST request requires: `Authorization: Bearer <bot_token>`, `AuthorizationType: ilink_bot_token`, and a random `X-WECHAT-UIN` header. All request bodies include `base_info: { channel_version: "2.0.0" }`.

The most critical concept is `context_token` вҖ” it's **not optional**. Without it, replies cannot be routed to the correct WeChat conversation. The SDK caches it per `(accountId, userId)` pair and persists it to storage.

<Section id="security" title="Security & privacy">

Everything runs **on your machine**. No remote servers, no accounts, no telemetry.

- **Local only** вҖ” all API calls go directly from your machine to `ilinkai.weixin.qq.com`
- **Credentials encrypted at rest** вҖ” stored with `0600` permissions in `~/.wechatbot/`
- **No proxy or relay** вҖ” the SDK communicates directly with WeChat's iLink servers
- **Media encryption** вҖ” all CDN uploads/downloads use AES-128-ECB; files are encrypted locally before upload
- **Session isolation** вҖ” each bot instance manages its own credentials and state independently

</Section>
