A **pi extension** that bridges WeChat to your [Pi coding agent](https://github.com/badlogic/pi-mono). Scan a QR code in WeChat → chat with Pi from your phone. Uses the `@wechatbot/wechatbot` SDK for all iLink protocol operations.

<Section id="how-it-works" title="How it works">

<CodeBlock lang="diagram" lineHeight="1.3" showLineNumbers={false}>
{`┌──────────────┐      ┌──────────────┐      ┌──────────┐
│  WeChat User │ ←──→ │  iLink API   │ ←──→ │ Pi Agent │
│  (phone)     │      │  (Tencent)   │      │ (laptop) │
└──────────────┘      └──────────────┘      └──────────┘
                              ↑
                   @wechatbot/wechatbot SDK`}
</CodeBlock>

1. You run `/wechat` in pi
2. Extension creates a `WeChatBot` instance (from the SDK) → calls `bot.login()`
3. SDK calls iLink API → gets a QR code URL → displayed in pi TUI
4. You scan the QR with WeChat on your phone
5. **WeChat messages → `pi.sendUserMessage()`** → become pi prompts
6. **`pi.on('agent_end')` → `bot.reply()`** → sent back to WeChat
7. Typing indicator shown while Pi thinks

</Section>

<Section id="install" title="Install">

**Option 1:** Load directly with pi:

```bash
pi -e /path/to/wechatbot/pi-agent/src/index.ts
```

**Option 2:** Copy to extensions directory (auto-loads):

```bash
# Global
cp -r pi-agent/ ~/.pi/agent/extensions/wechat-bridge/

# Project-local
cp -r pi-agent/ .pi/extensions/wechat-bridge/
```

</Section>

<Section id="message-flow" title="Message flow">

<CodeBlock lang="diagram" lineHeight="1.3" showLineNumbers={false}>
{`WeChat user sends "帮我重构 auth 模块"
         │
         ▼
  iLink API (long-poll getupdates)
         │
         ▼
  Extension receives IncomingMessage
         │
         ├── bot.sendTyping(userId)        → "对方正在输入中..."
         │
         ├── pi.sendUserMessage(msg.text)  → becomes pi prompt
         │
         ▼
  Pi agent processes (tools, thinking...)
         │
         ▼
  pi.on('agent_end') fires
         │
         ├── Extract assistant text
         ├── bot.stopTyping(userId)
         └── bot.reply(msg, text)          → sent back to WeChat`}
</CodeBlock>

</Section>

<Section id="commands" title="Commands">

```
/wechat              Start login (show QR code)
/wechat --force      Force re-login (new QR code)
/wechat-disconnect   Disconnect WeChat
/wechat-send <text>  Manually send text to WeChat user
```

When already connected, `/wechat` shows a menu: Disconnect, Show status, or Cancel.

</Section>

<Section id="features" title="Features & limitations">

**Features:**
- Zero config — just `/wechat` and scan
- Credential persistence — `~/.wechatbot/`, skips QR on restart
- Session recovery — auto re-login on `-14`
- Typing indicator — "对方正在输入中" while Pi thinks
- Text chunking — long replies split at 2000 chars
- Graceful shutdown — stops on `session_shutdown`

**Limitations:**
- Text messages only (images/voice/files acknowledged but not processed as prompts)
- Single WeChat user at a time
- QR displayed as URL (not rendered as actual QR image in terminal)
- Requires pi in interactive mode

</Section>
