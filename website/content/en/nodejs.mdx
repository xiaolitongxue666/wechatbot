The Node.js SDK is the most feature-rich implementation. **42 source files**, **10 modules**, **41 tests**, **zero runtime dependencies**. Requires Node.js ≥ 22.

<Section id="install" title="Install">

```bash
npm install @wechatbot/wechatbot
```

</Section>

<Section id="quick-start" title="Quick start">

```typescript
import { WeChatBot } from '@wechatbot/wechatbot'

const bot = new WeChatBot()
await bot.login()

bot.onMessage(async (msg) => {
  await bot.sendTyping(msg.userId)
  await bot.reply(msg, `Echo: ${msg.text}`)
})

await bot.start()
```

</Section>

<Section id="middleware" title="Middleware">

Express/Koa-style composable middleware pipeline. Runs before message handlers.

```typescript
import {
  WeChatBot, loggingMiddleware, rateLimitMiddleware,
  typeFilterMiddleware, filterMiddleware,
} from '@wechatbot/wechatbot'

const bot = new WeChatBot()

bot.use(loggingMiddleware(bot.logger))
bot.use(rateLimitMiddleware({ maxMessages: 10, windowMs: 60_000 }))
bot.use(typeFilterMiddleware('text', 'image'))
bot.use(filterMiddleware(/^\/\w+/))

// Custom middleware
bot.use(async (ctx, next) => {
  const start = Date.now()
  await next()
  console.log(`Processed in ${Date.now() - start}ms`)
})
```

Middleware that doesn't call `next()` stops the chain — perfect for auth, filtering, routing.

</Section>

<Section id="media" title="Media handling">

Full image/file/voice/video support with AES-128-ECB CDN crypto.

```typescript
bot.onMessage(async (msg) => {
  if (msg.type === 'image' && msg.images[0]?.media) {
    const data = await bot.downloadMedia(msg.images[0].media, msg.images[0].aeskey)
    await writeFile('photo.jpg', data)
  }

  if (msg.type === 'voice' && msg.voices[0]?.text) {
    await bot.reply(msg, `You said: "${msg.voices[0].text}"`)
  }
})
```

</Section>

const msg = bot.createMessage(userId)

<Section id="structure" title="Project structure">

```
nodejs/src/
├── core/           ← WeChatBot client, typed events, error hierarchy
├── transport/      ← HTTP client with retry & timeout
├── protocol/       ← Raw iLink API calls + wire types
├── auth/           ← QR login + credential persistence
├── messaging/      ← Poller, Sender, Typing, ContextStore
├── media/          ← AES-128-ECB crypto, CDN upload/download
├── middleware/      ← Express-style engine + 4 builtins
├── message/        ← Parser, Builder, friendly types
├── storage/        ← Pluggable (file, memory, interface)
├── logger/         ← Structured leveled logging
└── index.ts        ← Public exports
```
