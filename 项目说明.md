# WeChatBot 项目说明

## 项目概述

**WeChatBot** 是一个多语言、模块化、生产级的 SDK，用于基于**微信 iLink Bot API** 构建机器人。它能让任何 AI 智能体或应用通过 iLink 协议与微信用户通信。

本项目的目标是提供 "5 分钟接入" 体验，把任意的 AI Agent 接入微信。

- **仓库地址：** `github.com/corespeed-io/wechatbot`
- **开源协议：** MIT
- **状态：** 全部 4 个语言 SDK 均标记为 "生产就绪"

## 架构设计

所有 SDK 共享统一的分层架构：

```
应用层（你的 Bot 代码）
  → [中间件 (仅 Node.js)]
    → Bot 客户端（编排器：登录、运行、回复）
      ├── Poller      （长轮询 GET /getupdates）
      ├── Sender      （发送消息 POST /sendmessage）
      ├── Typing      （"正在输入..." 状态）
      └── Media       （CDN 上传/下载）
        → Context Store （context_token 生命周期管理）
          → Protocol / API （原始 iLink HTTP 调用）
            → Transport / HTTP （带重试的 HTTP 客户端）
              → Storage （凭证和状态持久化）
```

### 核心 iLink API 端点

| 端点 | 用途 |
|------|------|
| `GET /ilink/bot/get_bot_qrcode` | 获取登录二维码 |
| `GET /ilink/bot/get_qrcode_status` | 轮询扫码状态 |
| `POST /ilink/bot/getupdates` | 长轮询获取消息（35s 超时） |
| `POST /ilink/bot/sendmessage` | 发送文本/媒体消息 |
| `POST /ilink/bot/getconfig` | 获取输入状态票据 |
| `POST /ilink/bot/sendtyping` | 发送/取消输入状态 |
| `POST /ilink/bot/getuploadurl` | 获取 CDN 上传地址 |
| CDN 上传/下载 | 通过 `novac2c.cdn.weixin.qq.com/c2c` |

## 各语言实现详情

### 1. Go SDK（`golang/`）

- **模块：** `github.com/corespeed-io/wechatbot/golang`
- **要求：** Go 1.22+，零 CGO 依赖
- **核心文件：**
  - `types.go` — 所有公共类型定义（MessageType、CDNMedia、IncomingMessage、Credentials 等）
  - `bot.go` — 主客户端：New()、Login()、OnMessage()、Reply()、Send()、SendMedia()、Download()、Upload()、Run()、Stop()
  - `internal/protocol/api.go` — 原始 iLink HTTP 调用、鉴权头、API 错误处理
  - `internal/auth/login.go` — 二维码登录流程、凭证持久化至 `~/.wechatbot/credentials.json`
  - `internal/crypto/aes.go` — AES-128-ECB 加解密（PKCS7 填充）、密钥编解码
- **示例：** `examples/echo-bot/main.go`

### 2. Node.js SDK（`nodejs/`）

- **包名：** `@wechatbot/wechatbot` v2.0.1
- **要求：** Node >= 22，零运行时依赖
- **模块结构：**
  - `src/core/` — 客户端（统一回复/发送 API）、事件系统（类型化 EventEmitter）、错误类
  - `src/transport/` — 带重试的 HTTP 客户端
  - `src/protocol/` — 线类型 + iLink API 调用
  - `src/auth/` — 二维码登录 + 凭证持久化
  - `src/messaging/` — 轮询器、发送器、输入状态服务、上下文存储
  - `src/media/` — AES-128-ECB 加解密、CDN 上传/下载、MIME 路由、SILK→WAV 语音转码、远程 URL 下载、Markdown 清理
  - `src/middleware/` — Express/Koa 风格可组合中间件引擎 + 4 个内置中间件（日志、限速、类型过滤、消息过滤）
  - `src/message/` — 消息解析器、MessageBuilder（链式 API：`.text().image().file().build()`）
  - `src/storage/` — 可插拔存储（文件、内存或自定义如 Redis/SQLite）
  - `src/logger/` — 结构化分级日志

### 3. Python SDK（`python/`）

- **包名：** `wechatbot-sdk` v0.1.0
- **要求：** Python >= 3.9，依赖 aiohttp、cryptography
- **模块结构：**
  - `client.py` — WeChatBot 类：login、start、reply、reply_media、send、send_media、download、upload、send_typing、stop_typing
  - `types.py` — 所有数据类和 IntEnum 类型
  - `protocol.py` — 原始 iLink API 调用（aiohttp）
  - `auth.py` — 二维码登录 + 凭证持久化
  - `crypto.py` — AES-128-ECB 加解密 + 密钥编解码
  - `errors.py` — 错误层次（WeChatBotError、ApiError、AuthError 等）
- **构建工具：** hatchling
- **测试：** pytest（10 个加密测试 + 8 个客户端测试）

### 4. Rust SDK（`rust/`）— 功能最丰富

- **包名：** `wechatbot` v0.1.0
- **要求：** tokio + reqwest + serde
- **模块结构：**
  - `bot.rs` — 主客户端
  - `types.rs` — 所有类型（serde 序列化）
  - `error.rs` — 错误层次（thiserror）
  - `protocol.rs` — 原始 iLink API 调用
  - `crypto.rs` — AES-128-ECB 加解密
  - `session.rs` — **多机器人会话管理器**（状态：PendingQr、WaitingConfirm、Online、Expired、Offline）
  - `runtime.rs` — **MultiBotRuntime** — 编排多个机器人会话
  - `ingest.rs` — 事件规范化管道（消息 → Postgres + 媒体存储）
  - `forwarder.rs` — HMAC 签名的事件转发至 webhook 端点（带重试 + 死信队列）
  - `queue.rs` — 事件队列抽象（内存/Redis）
  - `config.rs` — TOML 配置 + 环境变量覆盖
  - `storage/` — PostgreSQL 聊天仓库、Redis 会话状态、LocalFs/S3 媒体存储
  - `admin/` — Axum 管理 Web 服务器（二维码展示、会话管理 UI）
  - `bin/admin.rs` — 管理端二进制入口
- **基础设施：** PostgreSQL 16、Redis 7、MinIO（S3 兼容存储）
- **Docker Compose：** `docker-compose.dev.yml`、`docker-compose.test.yml`

## 核心功能（所有 SDK 均支持）

| 功能 | 说明 |
|------|------|
| 二维码登录 | 凭证持久化至 `~/.wechatbot/` 目录，三步流程：获取二维码 → 轮询状态 → 确认登录 |
| 长轮询 | 35 秒服务端挂起连接，自动游标管理 |
| 富媒体支持 | 图片、文件、语音、视频（通过 CDN 上传/下载） |
| context_token | 自动从入站消息提取，通过 `reply()` 自动注入出站消息 |
| 输入状态 | "正在输入..." 提示，带票据缓存 |
| CDN 加密 | AES-128-ECB 加密，支持双密钥格式（直接 hex + base64） |
| 会话恢复 | 检测 `errcode: -14`（会话过期）后自动重新登录 |
| 文本分块 | 在 2000 字符处按自然边界（段落→行→空格）切分 |
| 文件自动路由 | 根据扩展名归类（`.png`→图片，`.mp4`→视频 等） |
| 指数退避 | 网络错误时 1s → 最大 10s 的指数退避重试 |

## Pi Agent 扩展（`pi-agent/`）

- **包名：** `@wechatbot/pi-agent` v0.1.2
- **用途：** 将 Pi AI 编程助手桥接到微信
- **功能：**
  - 注册 `/wechat` 和 `/weixin` 命令
  - 通过 `qrcode-terminal` 在终端显示登录二维码
  - 捕获 Pi 的 `agent_end` 事件，将清理后的文本发送回微信
  - 处理所有媒体类型：文本、图片（作为视觉输入）、文件（读取文本文件）、视频（保存到临时目录）、语音
  - 自动检测 Pi 输出中的媒体路径并作为微信附件发送
  - 注入系统提示词，告知 AI 当前已桥接到微信（无 Markdown，简洁回复）

## 设计模式

- **分层架构：** 传输层 → 协议层 → 服务层 → 应用层（所有 SDK 一致）
- **Context Token 模式：** 自动管理用户对话路由令牌
- **装饰器模式：** Python 的 `@bot.on_message` 注册处理器
- **建造者模式：** Node.js 的 MessageBuilder 构建消息
- **中间件模式：** Node.js 的中间件链处理消息管道
- **策略模式：** 可插拔存储后端（Node.js）和数据库模式（Rust）
- **观察者模式：** 生命周期钩子事件发射器（扫码、错误、会话过期）
- **熔断器/重试：** 所有 SDK 的网络故障指数退避
- **仓库模式：** Rust 的存储 trait（ChatRepository、SessionStateRepository、MediaStore）
